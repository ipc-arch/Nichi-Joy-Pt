# ニチジョイ🌸ポイント

日本女子体育大学の学内感謝ツール。タップだけで「ありがとう」「いいね！」「最高！」を匿名で贈れます。贈るたびに花が咲き、みんなの花畑が育っていきます。

---

## 🌷 概要

| 項目 | 内容 |
|------|------|
| 対象 | 日本女子体育大学 教職員・学生・附属校 |
| 機能 | 匿名の感謝メッセージを贈り合う |
| 費用 | 無料（GitHub Pages + Firebase無料枠） |
| 端末 | スマホ・PC・タブレット（ブラウザがあればOK） |
| インストール | 不要（URLにアクセスするだけ） |

---

## 🌸 使い方

1. **QRコードをスキャン** or URLにアクセス
2. **どこの？** → 部署・学科・部活を選ぶ
3. **だれに？** → 「あの人」「先輩」「さるお方」など匿名で選ぶ
4. **きもち** → ありがとう🌸 / いいね！🌻 / 最高！🌺
5. **花をおくる** → 花畑に花が咲く！

送り主の名前は「通りすがりのくろうにん」「隠れファンの応援団」など自動でランダム生成されます。

---

## 🔒 安全設計

- **テキスト入力フィールドなし** — 全操作が選択式。誹謗中傷・不適切入力の余地がゼロ
- **個人名なし** — 送り手・受け手ともに匿名ラベルのみ
- **データ** — 組織名（公開情報）+ 匿名ラベル + メッセージ種別 + タイムスタンプのみ保存
- **外部公開** — URL自体は公開だが、個人情報を一切含まないため漏洩リスクなし

## 🏗️ 技術構成

```
┌──────────────────┐     ┌──────────────────┐
│  GitHub Pages    │     │  Firebase        │
│  (フロントエンド)  │ ──→ │  Realtime DB     │
│  単一HTML        │     │  (共有花畑データ)  │
│  無料ホスティング  │     │  無料枠（Spark）   │
└──────────────────┘     └──────────────────┘
```

- **フロントエンド**: 単一HTMLファイル（React + インラインCSS）
- **バックエンド**: Firebase Realtime Database（認証不要・匿名読み書き）
- **ホスティング**: GitHub Pages（SSL自動・カスタムドメイン対応）
- **サーバ管理**: 不要

## 🔧 メンテナンス

### 組織・部活の変更

`index.html` 内の `ORG_GROUPS` 配列を編集：

```javascript
// 例: 新しい部活を追加
{
  group: "運動部",
  items: [..., "新しい部活名"],
}

// 例: 部署名を変更
// "旧名称" → "新名称" をテキスト置換
```

編集後、GitHubにpush → 数分で自動反映。

### 年度リセット

1. Firebase Console → Realtime Database を開く
2. データを削除（またはエクスポートしてからクリア）
3. 新年度スタート

もしくはアプリ内のリセットボタン（画面最下部）を使用。

### メッセージ種別の追加

`MESSAGE_TYPES` 配列に追加：

```javascript
{ id: "ganbare", label: "がんばれ！", emoji: "💪", colors: ["#34D399", "#10B981", "#059669", "#047857"] },
```

### 送り主・宛先ラベルの追加

- 送り主: `FROM_PREFIXES` / `FROM_SUFFIXES` 配列に追加
- 宛先: `TO_LABELS` 配列に追加

## 🚀 初期セットアップ手順

### 1. GitHubリポジトリ作成

```
リポジトリ名: nichijoy（推奨）
公開設定: Public
GitHub Pages: Settings → Pages → Branch: main
```

### 2. Firebaseプロジェクト作成

1. https://console.firebase.google.com/ にアクセス
2. 「プロジェクトを追加」
3. プロジェクト名: `nichijoy`（任意）
4. Realtime Database → 「データベースを作成」
5. セキュリティルール:

```json
{
  "rules": {
    "messages": {
      ".read": true,
      ".write": true
    }
  }
}
```

6. プロジェクト設定からWeb APIキーとdatabaseURLをコピー
7. `index.html` の Firebase設定部分に貼り付け

### 3. デプロイ

```bash
git add index.html
git commit -m "🌸 ニチジョイ初回デプロイ"
git push origin main
```

### 4. 学内展開

- QRコード付きチラシを掲示（同梱のチラシPDF使用）
- 学内メールでURL配信
- 学生ポータルにリンク掲載

## ❓ トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| 花が表示されない | Firebase接続エラー | ブラウザのコンソールでエラー確認 |
| 送信できない | ネットワーク接続 | Wi-Fi接続を確認 |
| 組織名が古い | HTML未更新 | ORG_GROUPSを編集してpush |
| 全員の花が見えない | Firebaseルール | `.read: true` を確認 |
